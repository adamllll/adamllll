# Product

## Register

brand

## Platform

web

## Users

Public GitHub visitors who encounter `adamllll` through the profile page. They are scanning quickly, often on mobile, and do not need a resume, project catalog, or technical identity statement.

## Product Purpose

Turn the profile README into a memorable, self-updating visual identity. Success means the page feels authored and alive while remaining concise enough to hand off naturally to GitHub's native contribution and repository modules.

## Positioning

The profile is a living identity where 52 weeks of real GitHub activity shape the name ADAM inside a night-mode ADAM_OS window.

## Conversion & proof

- Primary CTA: open `https://home.adamllll.com/` by clicking the profile poster.
- Secondary navigation: use the compact `SYSTEM DIRECTORY` rail for Projects, Blog, and About links before GitHub's native profile content.
- The line a visitor remembers after 10 seconds: 52 weeks of activity shape ADAM.
- Belief ladder: the activity is real, the visual updates automatically, the system boot is intentional, and the personal website is available through the integrated OS entry.
- Proof on hand: 52 weeks of contribution data, a daily GitHub Actions refresh, and a data-driven SVG generator.

## Brand Personality

Bold, kinetic, restrained. The profile should feel like a precise night-city signal rather than a conventional developer portfolio.

## Anti-references

- Badge walls, skill clouds, project cards, and filler sections added only to make the profile look complete.
- Generic hacker terminals, constant glitching, excessive neon glow, or disconnected cyberpunk decoration.
- Dashboard layouts that duplicate GitHub's native contribution graph.
- Compositions where ADAM is decoration beside a separate contribution visualization.
- A separate website panel or CTA that competes with the ADAM word mark.

## Design Principles

- Make ADAM and the contribution visualization the same object.
- Use the night-mode Win95 title bar and taskbar as structure, not as a second content section.
- Use real activity data as both content and composition.
- Orchestrate one strong boot sequence, then settle into a quiet title bar, taskbar, and ambient activity motion.
- Keep every supporting label functional and subordinate.
- Preserve a concise README with no fabricated proof or identity claims.

## Accessibility & Inclusion

Keep essential text at WCAG AA contrast, provide meaningful alternative text, scale the poster without horizontal scrolling, and disable all nonessential motion through `prefers-reduced-motion`.
